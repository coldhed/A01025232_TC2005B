Respuestas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una representación estructurada en forma de árbol de un documento HTML o XML, que permite acceder y manipular los elementos y contenido del documento mediante programación.

    Referencia
  2. ¿Qué es un nodo?

    3. Un nodo es un elemento individual en el árbol del DOM. Puede representar etiquetas HTML, texto, comentarios u otros tipos de elementos.

    Referencia
  3. ¿Cómo se accede a un elemento del DOM?

    4. Se puede acceder a un elemento del DOM utilizando métodos como getElementById, getElementsByClassName, getElementsByTagName o querySelector. Estos métodos devuelven referencias a los elementos que coinciden con los criterios de selección.

    Referencia
  4. ¿Cómo se modifica un elemento del DOM?

    5. Para modificar un elemento del DOM, se puede utilizar propiedades y métodos disponibles en el objeto del nodo. Por ejemplo, se puede cambiar el contenido de un elemento utilizando la propiedad innerHTML o modificar sus atributos con el método setAttribute.

    Referencia
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. getElementby* es un conjunto de métodos que permite seleccionar elementos del DOM según su ID, clase o etiqueta. Por otro lado, querySelector* es un método más flexible y potente que utiliza selectores CSS para seleccionar elementos.

    Referencia
  6. ¿Qué es un evento?

    7. Un evento es una acción o suceso que ocurre en un elemento del DOM, como hacer clic en un botón, mover el mouse, cargar una página, etc. Los eventos permiten la interacción del usuario y la ejecución de código en respuesta a esas acciones.

    Referencia
  7. ¿Qué es un listener?

    8. Un listener (o "escuchador") es una función que se vincula a un evento específico en un elemento del DOM. El listener está a la espera de que ocurra el evento y cuando eso sucede, se ejecuta el código asociado al mismo.

    Referencia
  8. ¿Qué es un callback?

    9. Un callback es una función que se pasa como argumento a otra función y se invoca dentro de ella en un momento específico o cuando se cumple una condición. En el contexto de los eventos del DOM, los callbacks se utilizan para definir la acción que se realizará cuando ocurra un evento.

    Referencia
  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento es un objeto proporcionado por el navegador cuando ocurre un evento en un elemento del DOM. Contiene información sobre el evento, como el tipo de evento, el elemento objetivo y otros datos relevantes. Se puede utilizar para acceder a detalles específicos del evento y realizar acciones en función de ellos.

    Referencia

Conclusiones

Experiencia general

Ya había usado ChatGPT y Copilot antes. Me gusta utilizarlos como herramientas de aprendizaje, no para reemplazarlo. Por ejemplo, con ChatGPT puedo tener conversaciones en las que le pregunto mis dudas y el me explica, e incluso en ocasiones le explico yo conceptos. Son herramientas que hay que aprender a utilizar pues son el futuro. No reemplazan a los programadores, pero son útiles para trabajar.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

ChatGPT cuando quiero pensar en cómo hacer algo. Usualmente no le pido código a ChatGPT, sino ideas de cómo hacer algo. Copilot lo utilizo para escribir código de manera más rápida. Por lo general son cosas que ya hice o ya se cómo hacer.

Ventajas de usar AI para programar

Las ventajas son que se pueden agilizar los procesos así como los procesos de aprendizaje.

Desventajas de usar AI para programar

Las desventajas es que pueden hacer que los desarrolladores se vuelvan flojos, y dejen de entender el código que producen, por lo que no se pueden hacer responsables por su código. ChatGPT y Copilot no reemplazan a los desarrolladores humanos ni al pensamiento crítico.
